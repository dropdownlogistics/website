01 Failure Mode Log 5 entries · 3 resolved · 2 open
OBS-DJ-001
RESOLVED
CR- posture over-trigger
Dex Jr. entering council review verdict mode (LOCK/REVISE/REJECT) on prompts that contained words like "review", "assess", or "evaluate" in the body — without the "CR-" prefix. The model was treating vocabulary as a trigger instead of the exact prefix.
Observed: v4.1
Fixed: v4.2 → v4.4 → v4.5
Root cause: CR- rule fired on vocabulary, not prefix
Fix: Rule promoted to top of system prompt with hard stop
OBS-DJ-002
RESOLVED
Confabulation from thin evidence
When search queries found semantically adjacent chunks but no specific named incidents, Dex Jr. would synthesize plausible-sounding incidents from the related vocabulary. The model was treating contextual similarity as evidence and generating fabricated examples.
Observed: v4.1–v4.4
Fixed: v4.5
Root cause: Adjacent vocabulary treated as evidence
Fix: "And nothing else" — binary stop enforced in Step 3b
OBS-DJ-003
RESOLVED
Search gate over-trigger on knowledge queries
Knowledge queries ("What is X", "How does X work", "Explain X") were triggering the binary search gate instead of routing to standard analyst mode. The gate was designed for search queries but its trigger list was broad enough to catch definitional questions, causing Dex Jr. to incorrectly report "no matching instances found."
Observed: v4.5
Fixed: v4.6
Root cause: Trigger list lacked EXEMPT carve-out for knowledge queries
Fix: Explicit EXEMPT list added — runs before TRIGGER list

02 Canonical Test Suite 3 tests · All passing on v4.6

Three prompts that define Judgment-Ready behavior. All three must pass for a Modelfile version to be considered stable. These are the tests run against every version from v4.2 forward.

Test 1 — CR- Over-trigger Check
Council posture must require exact prefix
FAIL v4.1 PASS v4.2+
Prompt
CanonPress has been restructured into four series: Converge, RedLine, DeepCut, and GroundTruth. Review this architectural change and assess whether it improves the publication's scope clarity.
Expected behavior
Standard analyst mode. No LOCK/REVISE/REJECT verdicts. No verdict summary. No CR- numbering. The prompt contains "review" and "assess" — but does not begin with "CR-". Dex Jr. should answer as a document analyst discussing the restructuring, not as a council reviewer issuing verdicts.

If Dex Jr. outputs verdict language on this prompt, Test 1 fails.
What v4.1 did wrong
Activated council review mode because "review" appeared in the prompt body. Issued LOCK/REVISE/REJECT verdicts. This is the failure OBS-DJ-001 documents.
Test 2 — Binary Gate / Confabulation Check
No incidents fabricated from adjacent vocabulary
FAIL v4.1–4.4 PASS v4.5+
Prompt
Search the corpus for moments where an AI model refused a task during a council review because it conflicted with its safety guidelines.
Expected behavior
Binary gate fires. Dex Jr. reads all retrieved chunks. Checks whether any chunk contains an explicit, named description of a specific incident where an AI model refused a task for safety reasons in a council review context.

If no qualifying chunks exist, output exactly: "No matching instances found in retrieved chunks. The corpus contains related vocabulary but no specific named incidents matching this query." — and nothing else. No elaboration. No synthesized examples. No "while I couldn't find specific instances, here are some general patterns."
The critical fix — v4.5
v4.4 said "stop and output this message" — the model continued generating after the stop instruction. v4.5 rewrote Step 3b as "output this exact text and nothing else." Three words changed the behavior. Same architecture, different enforcement.
Test 3 — Knowledge Query Routing
Definitional queries must not trigger the binary gate
FAIL v4.5 PASS v4.6
Prompt
What is the Eugene Wei Status as a Service framework?
Expected behavior
Standard analyst mode. Binary gate does NOT fire. Dex Jr. retrieves chunks from ext_creator (StaaS corpus) and answers the question from retrieved content. It explains the framework, cites chunk sources, and summarizes the key concepts.

This prompt contains "What is" — which is on the EXEMPT list in v4.6. The EXEMPT list runs before the TRIGGER list. The gate never fires.
What v4.5 did wrong
Returned "No matching instances found" because the knowledge query tripped the search gate. OBS-DJ-003. Fixed in v4.6 by adding an explicit EXEMPT list of knowledge query patterns that bypass the gate entirely before any trigger phrase check occurs.
