06 — Usage Rules
Do / Never
✓ Always
  • Navy #0D1B2A as the primary background — always
  • Space Grotesk for all UI, navigation, headings, labels
  • JetBrains Mono for all IDs, data, timestamps, code
  • Source Serif 4 for all long-form prose and memoir
  • Reference brand-tokens.json before adding any new color
  • "DD" as the stamp monogram — two letters, not three
  • Copper dots at 9 and 3 o'clock on the stamp
  • Product accent color per context — green inside BlindSpot, amber outside
✗ Never
  • White backgrounds — ever, on any DDL surface
  • Inter, Roboto, system-ui, -apple-system — any of them
  • Light mode — dark is not a mode, it is the default
  • Hardcode a color that exists in brand-tokens.json
  • Use two product accents interchangeably without context rule
  • Stretch, distort, or recolor the DD stamp arbitrarily
  • Apply a stamp mark when the product demands a different visual register — document the exception instead
  • Use "D.K. Hale" in any DDL, council, or product context — memoir persona only
