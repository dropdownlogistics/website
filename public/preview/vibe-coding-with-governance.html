DDL Build Methodology

Vibe Coding with Governance

How a one-person studio ships production sites at 2am
What is this?

A build methodology where the operator describes intent, an AI partner generates code and architecture, and the two iterate through rapid screenshot-feedback loops until the thing is shipped. The speed of vibe coding, the rigor of an audit practice.

Standard Vibe Coding

  • Describe what you want
  • AI generates code
  • Copy-paste and hope
  • No naming convention
  • No version history
  • Ship and forget

DDL Vibe Coding

  • Describe what you want
  • AI generates code + routes + manifests
  • One-line PowerShell deploy
  • Ratified naming standards
  • Git commits + session transcripts
  • Ship, inventory, iterate, document
The Loop
Six steps. One cycle. Repeat until shipped.
1

Describe

OPERATOR

The operator describes intent in plain language. Not a spec — a direction. "Three wings. One cathedral." or "Make it round with a crimson ring." Precision emerges through iteration, not upfront specification.

voice message / text / screenshot / handoff doc
2

Generate

CLAUDE

The AI partner generates complete, deployable artifacts: components, routes, deploy scripts, page factories. Not snippets — whole systems. Every file named, every route mapped, every dependency resolved.

.jsx / .tsx / .html / .ps1 / page.tsx routes
3

Deploy

OPERATOR

One-line execution. Download, copy to site directory, run the script. The operator doesn't debug code — they execute a deploy command and refresh a browser. PowerShell on Windows, terminal on Mac.

powershell -ExecutionPolicy Bypass -File deploy.ps1
4

Screenshot

OPERATOR

The operator sends back what they see. Not a bug report — a photo of reality. The screenshot is the truth. "This is what it looks like." The AI partner reads the pixels, not the code.

browser screenshot / terminal output / error paste
5

Iterate

BOTH

One-line surgical edits. No full rebuilds. "Make it bigger." "Add a crimson ring." "Make it round." Each adjustment is a single PowerShell command that targets one property. The loop tightens until it's right.

$content.Replace('old', 'new') | Set-Content
6

Ship + Record

BOTH

Git commit. Inventory update. Route manifest amended. Session transcript preserved. The governance layer catches what vibe coding usually drops: what was built, where it lives, and what it connects to.

git commit -m "feat: description" / manifest update

The Governance Layer
What makes it DDL, not just vibes

Every vibe coding session produces artifacts. DDL sessions also produce structure.

Naming

STD-0066

Route naming convention ratified by nine-model council. Every page gets a predictable path.

Templates

Six Molds

CONSOLE, CHRONICLE, ATLAS, LEDGER, CODEX, DOSSIER. Every page fits a template type.

Inventory

Route Manifest

Every deployed page tracked: file, route, wing, template, status. Nothing ships unaccounted.

Architecture

Wing System

DDL, D&A, DexVerse, Private, CottageHumble. Content organized by domain, not by date.

History

Session Transcripts

Every build session preserved. Context survives compaction. The build story is the documentation.

Versioning

Git + Branches

Feature branches for major changes. Commits with descriptive messages. Push when green.
