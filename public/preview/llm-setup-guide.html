The Build
10 Steps. One Inference Engine.
Follow in order. Don't skip ahead. Each step confirms the last one worked before moving on.
01

Install Ollama

Download from ollama.com. Run the installer. It adds a system tray icon and a background service. That's it — no configuration needed.

download
# Go to: https://ollama.com/download/windows # Run the .exe. Click through. Done.
⏱ ~2 minutes
02

Verify Installation

Open PowerShell (not as admin — just normal) and confirm Ollama is running.

powershell
ollama --version # Expected: ollama version 0.x.x
⏱ ~30 seconds
03

Pull Your Primary Model

This downloads the model weights to your machine. ~4--6 GB depending on the model. One command.

powershell — primary model
ollama pull qwen2.5-coder:7b # ~5.8 GB download. Structured output, code gen, schema compliance.
⏱ ~5 minutes (depends on connection)
04

First Inference

Talk to your model. If this works, your GPU is running inference locally. Check Task Manager → Performance → GPU to see VRAM spike.

powershell — first run
ollama run qwen2.5-coder:7b "Hello from my local GPU" # You should get a coherent response in ~2 seconds. # Type /bye to exit interactive mode.
⏱ ~30 seconds — the moment it becomes real
05

Test the API Endpoint

Ollama exposes an OpenAI-compatible API on localhost. This is what your scripts will talk to.

powershell — api test
curl http://localhost:11434/api/generate -d '{ "model": "qwen2.5-coder:7b", "prompt": "Say hello", "stream": false }' # Returns JSON with the model's response.
⏱ ~1 minute
06

Run Calibration Protocol

Six prompts that test whether your model is ready for real work. Compliance, schema output, code generation, intent evaluation, persona simulation, and context comprehension. Pass threshold: 5/6 clean.

⏱ ~10 minutes
07

Pull Secondary Model

Your second opinion. Different training, different blind spots. When two models agree, that's signal.

powershell — secondary
ollama pull llama3.1:8b-instruct # ~5.5 GB. Reasoning, general capability, convergent analysis.
⏱ ~5 minutes
08

Pull Compact Model

Lightweight and fast. Use for simple parsing, quick extractions, or when you need speed over depth.

powershell — compact
ollama pull phi3.5 # ~2.2 GB. 128K context. Fast inference. Document processing.
⏱ ~3 minutes
09

Write Your First Bridge Script

A Python script that calls the Ollama API, passes your data, and returns structured output. This is where the model meets your workflow.

python — bridge.py
import requests, json response = requests.post( "http://localhost:11434/api/generate", json={ "model": "qwen2.5-coder:7b", "prompt": "Your structured prompt here", "stream": False } ) result = json.loads(response.text) print(result["response"])
⏱ ~15 minutes
10

Confirm Operational

Three models loaded. API responding. Calibration passed. Bridge script working. Your gaming rig is now an inference engine.

powershell — verify all models
ollama list # Should show all 3 models with sizes.
✓ You're operational.
Quality Gate
Calibration Protocol
A local model without calibration is a toy. These six prompts prove your model is ready for real work. Pass 5 of 6.
TEST 01 — COMPLIANCE

Exact Output

"Respond with exactly: Hello from Dex Jr."
Tests: Can the model follow a precise instruction?

TEST 02 — SCHEMA

Structured JSON

"Output valid JSON with all 18 schema fields."
Tests: Schema compliance and structured output.

TEST 03 — CODE

Formula Generation

"Write a MAKEARRAY + LET + LAMBDA formula. No explanation."
Tests: Code generation without commentary.

TEST 04 — INTENT

Grammar Evaluation

"Evaluate this intent against A-001 grammar. PASS or FAIL with fix."
Tests: Analytical judgment.

TEST 05 — PERSONA

Architectural Review

"You are a council member reviewing a decision to use Power Query as search engine."
Tests: Role adoption and domain critique.

TEST 06 — CONTEXT

Spec Comprehension

"What are the three edge types in tbl_Edges?"
Tests: Document comprehension from full spec paste.

Architecture
How to Think About Local AI
→
The LLM interprets. It doesn't compute. Numbers come from your engine — spreadsheets, databases, fact tables. The model reads them and tells you what they mean. Never trust a local model to do math.
→
Convergent > singular. Two models agreeing on the same flag = high-confidence signal. Where they diverge = your judgment call. Run the same prompt through both and compare.
→
Prompt templates are the contract. Structure beats intelligence. A well-governed prompt with a mediocre model outperforms a sloppy prompt with a great model. Write your prompts like you write your schemas.
→
Local ≠ better thinking. Local gives you privacy, control, and cost stability. It does not give you epistemic superiority. Cloud models are still smarter on hard reasoning. Know when to use which.
→
Models are swappable. The engine doesn't care. Ollama lets you pull new models in one command. Your prompts, your API calls, your bridge scripts — none of them change when the model changes. That's the architecture.
→
Constraints are features. 8GB VRAM isn't a limitation — it's a filter. It told us exactly which models fit and which don't. Embrace the constraint. Let it do the thinking.
Cottage — Humble surface. Cathedral underneath.
