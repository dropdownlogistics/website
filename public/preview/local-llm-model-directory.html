Philosophy
How to Choose
→ Start with one model. Get it working. Get it calibrated. Get it integrated into a real workflow. Then add a second for convergent analysis. Model hoarding on Day 1 is procrastination dressed as preparation.
→ Match the model to the task. Code models for code. Reasoning models for analysis. Compact models for speed. Don't use a 7B general model for code when a 7B code model exists.
→ VRAM is the constraint. Disk is free. Download as many models as you want. Only one loads into VRAM at a time. 500GB of disk holds 50+ models. Your GPU holds one.
→ Calibrate before you trust. Every new model gets the 6-prompt calibration protocol before it touches real work. Compliance, schema, code, intent, persona, context. 5/6 pass threshold.
→ Models are swappable. Prompts are permanent. New models come out every month. Your governed prompt templates, bridge scripts, and system prompts outlast every model. Build the pipeline, not the model dependency.
