05 — Architecture
Star Schema + Privacy First
Data Model
  • Star schema: Fact_Stub at center
  • Dimensions: Dim_Event, Dim_Venue, Dim_User, Dim_Date
  • Schema does not branch by domain — data does
Verification Tiers
  • Verified — external API match
  • High Confidence — metadata correlation
  • Self-reported — user-entered only
Enrichment
  • Setlist.fm — concert setlists
  • TMDB — film metadata
  • Anniversary engine — stubs resurface with full context
Privacy Governance
  • STD-DDL-ANTIPAT-001 governing
  • No tracking pixels, no fingerprinting
  • No data monetization — revenue without surveillance
  • User owns data: full export, full deletion
